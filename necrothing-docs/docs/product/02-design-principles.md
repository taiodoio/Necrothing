# Design Principles

Versione: `v0.1`

## 1. Local-first by default

Ogni funzione deve funzionare offline. Le feature che richiedono server, account o sincronizzazione remota sono escluse dalla versione iniziale.

## 2. L'app non deve mendicare attenzione

Le notifiche devono essere narrative e disattivabili. Non devono sembrare marketing.

## 3. Ogni oggetto deve diventare una storia

Una tomba non è solo un record. Deve accumulare memoria: visite, fiori, anniversari, eventi, apparizioni, manutenzione.

## 4. Progressione estetica, non predatoria

L'app può usare XP, ranghi, achievement e sblocchi, ma non deve creare dipendenza tossica o pressione quotidiana aggressiva.

## 5. Nessuna monetizzazione invasiva

Niente ads, niente loot box, niente microtransazioni nella versione base. Eventuali monetizzazioni future devono essere cosmetiche, trasparenti e non necessarie.

## 6. Umorismo coerente

Il linguaggio deve essere ironico, funereo, teatrale e asciutto. Evitare meme troppo datati o battute eccessivamente rumorose.

## 7. Interazione semplice

L'app deve essere comprensibile senza tutorial lunghi: tocca una zolla, seppellisci, commemora, decora.
